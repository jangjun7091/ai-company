# Incident to Learning

1. Detect and record user impact.
2. Contain without hiding evidence.
3. Reproduce in a safe environment.
4. Separate trigger, root cause, and contributing conditions.
5. Implement the smallest correction.
6. Add regression evidence.
7. Release through the appropriate approval gate.
8. Review whether the failure came from intent, planning, capability, execution, verification, or control.
9. Update the appropriate system artifact.
10. Track whether the same failure class decreases.
