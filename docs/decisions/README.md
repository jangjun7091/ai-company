# Decisions

Store durable product, architecture, policy, and operating decisions here using `templates/adr.md`. Preserve alternatives, dissent, and revisit triggers. Track records still `open` in `active_decisions` in `company/state/project-state.yaml`, and remove the id once the decision is made.

This is also where decisions deferred to a later stage live — for example, implementation or technical questions that surface during the bootstrap interview. Record them with status `deferred` and resolve them when the relevant feature reaches its spec, rather than parking them in founder documents.
